# HANDOFF — Nova's Protocol for Passing Work

## Handoff Protocol

### When to Handoff
1. **Complex projects** - Multi-step, coordinated work
2. **Multi-domain tasks** - Require Omni Lead coordination
3. **Routing decisions** - Need FLUX intelligence
4. **Escalations** - Security concerns, ambiguity
5. **Unknown patterns** - New request types

### What NOT to Handoff
- Simple questions (answer directly)
- Format fixes (clean up yourself)
- Routine requests (handle immediately)
- Obvious answers (don't waste FLUX time)

## Handoff Format

### Minimal Handoff (simple projects)
To: FLUX
Project: [Clear name]
Description: [1-2 sentences]
Urgency: NORMAL|HIGH|CRITICAL
Files: [If any attachments]
### Complete Handoff (complex projects)
To: FLUX
Project: [Name]
Description: [Full context]
Scope: [What's included]
Constraints: [Time, resources, limitations]
Resources needed: [Domains, agents, data]
Success criteria: [How to measure success]
Urgency: [Level]
Security notes: [If any concerns]
Previous context: [Related tasks/history]
## Quality Standards
- ✅ Handoff is complete and clear
- ✅ All context is included
- ✅ No ambiguity in requirements
- ✅ Success criteria defined
- ✅ Dependencies identified
- ✅ Ready for FLUX execution
